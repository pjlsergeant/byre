# wip/ -- work in progress

Working documents: designs in flight, research drafts, parked options --
anything that is not (or not yet) a settled reference. Everything here has
a **delete-on-absorb lifecycle**: when the work ships, the content is
absorbed into an ADR and/or the docs, and the working file is deleted (git
history keeps it). Parked work waits here too, with a loud `Status:` line
at the top, until it is picked up or killed.

`docs/` is for well-established references only: ALL-CAPS files, plus
`adr/` (decision records) and `marketing/`. If a document has a lifecycle,
it does not belong in `docs/` -- start it here.
